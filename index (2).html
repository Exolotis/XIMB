Phase 02

How to Ask AI for Better Code

⏱ 15 minutes

If you just say "build me a website," the AI might give you messy or confusing results. We'll learn how to give clear, specific instructions so you get exactly what you want. You're free to use whichever AI assistant you prefer — the prompts below work the same way with any of them.

🧩 Tip: Bring the AI Into VS Code

Instead of copying and pasting between a browser and your editor, you can install an AI extension directly in VS Code. One popular option is the Codex extension — open the Extensions panel in VS Code (the squares icon on the left), search for it, and click Install. Other AI coding extensions work the same way. You have full freedom to pick whichever agent you like best; the steps in this guide don't depend on any single one.

2

Writing Great Instructions

The secret is giving the AI clear context: what you're building, the colours you want, and a request to keep things simple. Here are copy-paste prompts you can use right now — tap Copy, paste them into your AI, and tweak the details in brackets.

Watch: prompting AI to write codeOpens a YouTube search ↗
★ Starter Prompt
"I'm a complete beginner. Build me a single-file index.html for a personal portfolio. Use a dark theme with [green] accents. Keep all CSS and JS in one file, and add short comments explaining each section."
Adding a Feature
"Please add a section for [a contact form / photo gallery / about-me]. Match the existing colours and fonts, and show me exactly where to paste the new code."
Fixing Something
"Something looks wrong. Here is my full code below. Explain in simple terms what's broken and give me the corrected version: [paste code]"

⚠️ Common Beginner Mistakes

  • Wrong file names: When you copy code from the AI into VS Code, double-check that your file ends in the exact extension the AI told you to use (like .html or .jsx).
  • Missing pieces: Sometimes AI cuts off long code. Always make sure you copy the entire block!

Phase 03

Viewing Your Website on Your Computer

⏱ 10 minutes

This is the easiest step of all. Since the AI built you a single website file, you don't need any complicated setup to see it — you can open it straight in your browser, and use a free VS Code add-on to watch it update live as you make changes.

3

Previewing Your Website

First, save the code the AI gave you. In VS Code, create a file called index.html inside your project folder, paste in the full code, and save it with Cmd + S (Mac) or Ctrl + S (Windows). This video shows the whole save-and-preview routine:

Two easy ways to see your site:

  • The quick way: find your index.html file on your computer and double-click it. It opens in your web browser straight away.
  • The live way (recommended): in VS Code, install the free Live Preview extension, then right-click your index.html and choose "Show Preview." Now every time you save, the page refreshes by itself.

Pro Tip: Keep VS Code open on the left side of your screen and your browser on the right. With Live Preview turned on, saving a change in VS Code updates the page instantly!

⚠️ Common Beginner Mistakes

  • Wrong file name: The file must be named exactly index.html — not index.html.txt. In VS Code, the name you type is the real name, so you're safe there.
  • Forgetting to save: Your browser shows the last saved version. If a change isn't showing, press Cmd + S / Ctrl + S first, then refresh.

Phase 04

Putting Your Website Online

⏱ 20 minutes

Your website works perfectly on your computer. Now it's time to save it safely to GitHub, and use Vercel to give it a real, clickable link that anyone in the world can visit.

4

Saving to GitHub & Launching on Vercel

This final phase connects all the dots. You'll upload your files to GitHub (your safe locker), then point Vercel at them to host on the internet. Three moves:

  • Push to GitHub: create a free GitHub account, make a new repository, and upload your project folder.
  • Import into Vercel: sign in to Vercel with GitHub and select the repository you just created.
  • Deploy: click Deploy and wait about a minute for the build to finish.

The smoothest way to push your files is to connect GitHub directly to VS Code, so you can save to the cloud without leaving your editor. This short video shows how to link the two:

⚠️ Common Beginner Mistakes

  • Forgetting to Push: Saving a file in VS Code doesn't automatically send it to GitHub. You have to "commit" and "push" your changes first.
  • Connecting the wrong repo: When Vercel asks which project to publish, make sure you select the one you just created today!

⚠️ Common Beginner Mistakes

  • Editing on the wrong machine: Always make changes in your VS Code project folder — not directly on the GitHub website — so your computer and the cloud stay in sync.
  • Forgetting to push: Saving in VS Code only saves locally. The live site updates only after you commit and push to GitHub.
  • Refreshing too soon: Give Vercel a minute to rebuild. If the change isn't showing, wait a moment and do a hard refresh of your browser.

I'm Stuck. What Do I Do?

Take a deep breath. Every single developer runs into errors. Here is how to fix the most common issues beginners face.

🤖 The Ultimate Backup Plan: Don't Panic!

If you ever feel completely stuck and none of the quick fixes below are working, remember this golden rule: You can always take a screenshot. Just capture your whole screen (showing VS Code, the terminal, or the error message), upload the image to your AI, and ask: "I'm stuck here. What am I doing wrong?" The AI is incredibly good at reading errors and will guide you straight out of it.

My index.html opens as code/text instead of a web page
This usually happens if the file was saved as plain text. Make sure the file name ends in .html (not .txt). If you double-click it and it still opens in an editor, right-click the file, choose "Open With," and pick your web browser instead.
The Live Preview extension isn't showing anything
First make sure you installed the Live Preview extension from the Extensions panel in VS Code. Then right-click your index.html file and choose "Show Preview." If the panel is blank, close it and reopen it, and double-check you saved the file with Cmd + S / Ctrl + S.
My website isn't updating when I make changes
Your browser only shows the last saved version of the file. First, make sure you actually saved (Cmd + S on Mac, Ctrl + S on Windows). Then refresh your browser tab. If you're using Live Preview, saving should refresh it for you automatically.
I pasted the code but the page looks broken
Make sure you copied the entire block of code the AI gave you, from the very first line to the very last. Missing the top or bottom is the most common cause. If it still looks off, paste your full code back to the AI and ask: "Something looks broken — what did I miss?"
Vercel says my deployment failed
Check the red error message in Vercel. Often it's a small typo in your code or a file named incorrectly. You can copy Vercel's error message and paste it directly into your AI, asking: "How do I fix this error?"
